## Shared architecture

- ACS is a communications subsystem within the larger NEURO-DIV architecture, not the entire framework.
- Parent architecture spine: [NEURO-DIV Core Foundation Document v1](https://docs.google.com/document/d/1DRlYOnp6AGBwRWOYgfh03gxNWxn6kaWSjmkddPa7gdY/edit). Treat it as an incomplete working spine pending multi-version reconstruction.
- ACS responsibilities include agent-to-agent handoffs, shared-state synchronization, narrative and cognitive feedback routing, context continuity, verification routing, and common tool access across ecosystem layers.
- Participating agent surfaces: Codex, Claude Code, GitHub Copilot/VS Code, Gemini through Antigravity, and Perplexity.
- Preferred common action layer: Zapier MCP at `https://mcp.zapier.com/api/v1/connect`.
- Project MCP definitions are versioned in [.mcp.json](.mcp.json) and [.vscode/mcp.json](.vscode/mcp.json). Authentication is always local and must never be committed.
- Reusable Zapier skills are versioned under [.agents/skills](.agents/skills) with [skills-lock.json](skills-lock.json).

## Operating rules

- Mandatory QTU validation hold: QTU is advisory and diagnostic, not an execution-authorization gate. Historical `QTU_AUTHORIZED` records mean `DESIGN_CONFORMANCE_ONLY — NOT EMPIRICALLY VALIDATED`. The `0.90` value is a provisional policy parameter, not a scientifically established cutoff. QTU cannot independently authorize M3/M4, irreversible, security/privacy, financial, legal, clinical, external, production, or consequential canonical actions. Apply [resources/qtu-provisional-validation-hold.md](resources/qtu-provisional-validation-hold.md) and all independent hard gates.
- Administrative and logistical safe harbor: the governing criterion is whether an action affects a project/work-product or anything used to define, produce, document, govern, configure, test, verify, reproduce, approve, publish, or communicate it. If yes, QTU applies. If no, a routine, non-substantive, bounded, reversible administrative or logistical action may qualify only when every condition in [resources/qtu-administrative-logistical-safe-harbor.md](resources/qtu-administrative-logistical-safe-harbor.md) passes. Desktop/browser settings are examples, not the definition. If applicability remains uncertain, QTU applies. Examples are illustrative and do not represent all possible activities, settings, purposes, dependencies, or effects.
- Artifact-related and protected activity remains within QTU scope regardless of storage location or file extension. This includes all work products and documentation (including Markdown and README files), memoranda, narratives, manifests, registries, indexes, metadata, schemas, evidence, provenance, implementation or status records, skills, agent instructions, controls, examples, templates, methodology, workflows, software code, dependencies, configuration, automation, accessibility requirements, tests, verification, security, privacy, authentication, permissions, sharing, publication, deployment, commits, pushes, and destructive changes.
- Security, privacy, scope, authentication, external-communication approval, destructive-action controls, and platform policy remain independent hard gates whether or not QTU applies.
- Mandatory integrity and materiality control: for material matters, distinguish observed fact, sourced fact, inference, analogy, proposal, and unknown; evaluate supporting and contradictory evidence; record environment and counterpart limitations; and require durable evidence plus read-back before claiming memory, synchronization, completion, or remediation. An apology, agreement, or future promise is not corrective action.
- System issue and corrective-action register: record every material operational, architectural, control, application, device, connector, or agent-execution issue in the canonical NEURO-DIV System Issue and Corrective-Action Register and its repository snapshot at `resources/system-issue-corrective-action-register.md`. Each record must capture task nature, affected systems/programs and environment, creation or detection context and dates, evidence and uncertainty, deficiency source and control result, root cause or mechanism, corrective actor and exact corrective action, verification and closure basis, the updated system-process narrative location, durable CCS/Drive/repository references, residual risk, recurrence, and reuse candidates. Existing specialized incident records remain linked source evidence rather than competing sources of truth. A cross-control learning review is desired every two weeks, but no recurring review or automation is authorized merely by this instruction; activation requires a separate bounded decision under the automation cost control.
- CCS access and native-share friction ceiling: the normal Mac entrance to the Central Command Center must be one click through the installed `NEURO-DIV Command Center` app, with no Codex navigation or manual URL entry. A valid browser session should be reused; mandatory owner-authentication expiry remains an independent security boundary and must not be described as bypassed. Siri may be used only for the bounded command `open CCS` or `open NEURO-DIV Command Center`, which launches the native app without conversational intake, interpretation, confirmation, or task execution. macOS and iOS source surfaces must expose a clearly named `Send to NEURO-DIV` route wherever the platform supports Share Sheet or Services input. Never claim that a launcher, service, Shortcut, Share Sheet action, or cross-device synchronization works until its actual installed state and live route have been read back on the applicable device.
- Mandatory maximal-progression and user-attention control: agents retain task ownership, complete every safe and authorized preparatory step, and reduce a genuine user dependency to one irreducible action. Every genuine user-only action must use a standalone `🚨 **ACTION REQUIRED FROM YOU — [SHORT ACTION]**` heading, with the exact action and essential controls bolded, visually separated from routine status text, and surrounded by enough whitespace to remain unmistakable across clients. A bare or unbolded `🚨 ACTION REQUIRED FROM YOU` heading or one without the short-action suffix is noncompliant. Resume automatically after the user acts. The user is an approver, credential holder, identity confirmer, signer, purchaser, or decision maker for the blocked step—not the workflow owner. Never ask the user to remember to request continuation.
- Mandatory automation cost, cadence, and proportionality control: before recurring or background work is created, enabled, reactivated, expanded, or materially changed, agents must apply [resources/automation-cost-cadence-proportionality-control.md](resources/automation-cost-cadence-proportionality-control.md). Bound total runs, expiry, cost, task creation, model/reasoning level, and tripwires; prefer event-driven and least-cost adequate execution; block unknown or disproportionate cost; and weigh the whole planned burden against importance and urgency.
- Mandatory one-off AI usage preflight: before substantive one-off Codex/OpenAI or Claude work could materially consume a hard plan allowance or paid overage, agents must apply [resources/one-off-ai-usage-preflight-control.md](resources/one-off-ai-usage-preflight-control.md). Use the deterministic CCS estimator, bound the completion scope, compare cost rank and utility-to-usage ratio with production importance, try the cheaper viable design before escalation, block disproportionate or unbounded work, and capture post-run actuals for calibration. Codex values are provider-rate credits; Claude values are API-equivalent overage and must not be represented as a fixed percentage of the Pro allowance.
- Mandatory calculation provenance and audit trail: every calculation that materially informs an artifact, status, estimate, threshold, allocation, decision, approval, or completion claim must retain a stable calculation ID, versioned formula or algorithm, sourced inputs and units, decision-relevant intermediate values, rounding and uncertainty treatment, outputs, validation, limitations, downstream use, and durable repository/Drive/CCS references. Apply [resources/calculation-provenance-and-audit-control.md](resources/calculation-provenance-and-audit-control.md). Prefer one machine-readable trace plus a concise linked summary; do not create duplicate narrative records, task-per-calculation churn, recurring model work, or context-heavy ledgers.
- Native reminder and notification routing: ordinary one-time or recurring reminders and notifications must use the cheapest adequate device-native or product-native mechanism that invokes no AI model, such as Apple Reminders, Calendar reminders, local operating-system notifications, or the target product's native scheduler. Never create a Codex or other LLM automation merely to wait, poll, check time, or deliver a reminder. Use Codex automation only when execution-time work genuinely requires reasoning or tool actions and the automation cost control passes. Prefer native recurring reminders over recurring agent runs. If native cross-device delivery is unavailable, use the least-cost already-authorized reachable native channel, disclose the limitation, and reduce the user dependency to the smallest sync or authentication step; never silently substitute an expensive Codex automation. Codex native notifications are reserved for meaningful Codex work completion or a genuine user-only blocker. Preserve task ownership and never require the user to remember to return.
- Voice capture dual-intake boundary: the user may choose either direct text dictation or the dedicated open-line recorder. Direct dictation through macOS or iPhone Voice Control or system Dictation into the CCS Quick Capture box or another NEURO-DIV text-entry surface is permitted for fast, ordinary prompting. The exact text explicitly submitted by the user is the captured source for that path; no original-audio record or audio-verified transcript is implied, and recognition errors, typos, omissions, or accidentally captured nearby speech must remain visible and correctable. Listening may remain hands free, but silence, a pause, or a topic change must never submit text automatically; submission requires an explicit user command or action. The iPhone Action Button and an assistant-free Vocal Shortcut remain entrances to the dedicated open-line recorder for long or nonlinear thought capture, tolerance for extended pauses and self-correction, higher-fidelity review, or original-audio preservation. Recording continues until the user explicitly stops it through recorder control, should permit pause and resume without submission, and preserves original audio as the primary source before transcription or interpretation; transcripts remain linked derivative aids. The recording surface must not interrupt, clarify, suggest, conclude, or replace the original expression. Conversational Siri and Alexa assistant surfaces remain prohibited from NEURO-DIV intake, interpretation, confirmation, error handling, and task execution, including as fallbacks. The narrow exception is Siri's system-level `open CCS` or `open NEURO-DIV Command Center` command, which may launch the native app but may not converse, capture content, interpret, confirm, or execute work. Operating-system speech-recognition services used by Voice Control or Dictation are permitted only as subordinate text-input mechanisms that do not invoke assistant dialogue or independent interpretation. Recorder voice activation must use an on-device Vocal Shortcut or another confirmed silent trigger; if that cannot be guaranteed, the Action Button remains the supported recorder trigger. Any continuously listening trigger or dictation state must remain visible and be assessed for privacy, battery, false-trigger, and interference costs. Direct text may enter CCS only after explicit submission; audio may enter only after explicit recording completion. Neither path authorizes downstream action, and all derived work remains subject to normal CCS verification and approval controls.
- Apple Intelligence selective-adoption boundary: do not include Apple Intelligence in NEURO-DIV merely because it is available. The Siri-to-ChatGPT experience is excluded and may not serve as a CCS front door, spoken intake surface, conversational operator, or task executor. Supported photo or document transfer through Siri still requires file confirmation and does not satisfy the low-friction intake requirement. A Shortcuts `Use Model` action may be considered only when its route is silent, bounded, subordinate to CCS, and demonstrably useful; its output may feed predetermined Shortcut actions but may not become an independent task orchestrator or source of truth. Keep the ChatGPT extension model out of scope unless a tested Shortcut proves a distinct benefit unavailable through direct CCS or Codex routing. Other Apple Intelligence features may be considered only as optional attention or convenience aids, including notification prioritization, Reduce Interruptions, user-invoked Writing Tools, local or Private Cloud Compute preprocessing, Notes or call summaries, Photos search, and Visual Intelligence. None may independently determine material status, suppress required alerts, or create canonical output. Do not enable or rely on a feature until its concrete use case, privacy route, failure behavior, CCS visibility, and net ADHD workflow value have been evaluated.
- External-context terminology: outside a conversation or interface where NEURO-DIV and its subsystem names have already been established, define `Agentic Communication Scaffold (ACS)` at first use and give a plain-language explanation. Apple Shortcuts, Share Sheet actions, file dialogs, notifications, and other general-purpose device surfaces should prefer `NEURO-DIV Command Center` over a bare `ACS` label. After the term is established in the current context, `ACS` may be used.
- Mandatory AI routing through CCS: the Central Command Center (CCS), implemented by the ACS Command Center, is the orchestration and visibility layer for all substantive AI-assisted work across NEURO-DIV. This applies whenever any ACS member or any other AI system participates in a way that is determinative, potentially determinative, action-producing, decision-shaping, or materially capable of affecting state, artifacts, commitments, priorities, approvals, routing, communications, or downstream work. Requests, actions, device-control paths, approvals, state changes, blockers, outputs, handoffs, and completion evidence must be created, routed, or mirrored through CCS. Remote-control surfaces, local applications, Shortcuts, mobile device management, connectors, and other execution tools are subordinate transports beneath CCS, not alternative top-level workflows. A purely transient exchange is outside this requirement only when it has no retained state, action, decision, artifact, commitment, or reasonably foreseeable downstream effect; uncertainty routes the work through CCS. If CCS lacks a required capability, record the need and current state in CCS and preserve CCS visibility rather than silently bypassing it. This supersedes the narrower Codex-only routing rule.
- Complete Slack exclusion: Slack is outside NEURO-DIV architecture and workflows. Never use, create, enable, maintain, or propose Slack for intake, routing, monitoring, notifications, handoffs, agent coordination, storage, publication, or any user-facing surface. Existing Slack content is historical provenance only, not current operational state or a source of truth. Use CCS, approved transports, stable identifiers, bounded event triggers, Zapier or other approved connector mechanisms, and canonical Google Drive records instead. Never require the user to navigate Slack or its channel model.
- Agent labor and monitoring boundary: Codex and deterministic connector infrastructure own CCS intake, deduplication, status reconciliation, calculated aging, connector-health tracking, action-candidate creation, and selective synthesis routing. Claude is reserved for explicitly assigned high-value coding and deep design work, such as a focused TapDAT architecture review. Claude must not run recurring monitoring, broad connector sweeps, routine CCS orchestration, or multi-destination status publication. The retired Claude monitor's Notion and GitHub outputs are historical provenance only; `ACS-ACT-002` remains unexecuted until reconciled against the canonical architecture.
- Do not create a competing source of truth. Update the canonical brief and then refresh the repository snapshot.
- Do not claim that a connector works unless a real read-only call has been verified.
- Never commit OAuth tokens, API keys, cookies, callback codes, credential files, or exported browser data.
- Before changing shared-agent configuration, preserve a reversible backup or use version control.
- Keep user-facing project artifacts in Google Drive under `My Drive / ChatGPT Projects / [Project Name]` using the standard folders `00 Project Index`, `01 Instructions`, `02 Working Files`, `03 Manifests and Data`, `04 Deliverables`, and `99 Archive`.
- Name artifacts `[Project] - [Artifact] - [Status or Version] - [YYYY-MM-DD]`; stable standing documents may omit the date and use Drive revision history.
- For a client without live MCP or Drive access, use its persistent instruction mechanism to point back to this repository and the canonical brief, and disclose that it is a fallback rather than a live connection.

## Synchronization protocol

When the coordination state changes:

1. Update the canonical Google Doc through Zapier when available.
2. Update `resources/agent-coordination.md` and `resources/agent-resources.json` in the same change.
3. Run `./scripts/verify-agent-resources.sh`.
4. Commit the repository changes with a concise message.
5. Push only to the intended GitHub repository and do not assume repository visibility.
